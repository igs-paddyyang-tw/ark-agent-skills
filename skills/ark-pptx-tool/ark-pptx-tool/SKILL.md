## 設計靈感

**不要製作無聊的投影片。** 白色背景上的純文字條列不會讓任何人印象深刻。為每張投影片考慮以下清單中的想法。

### 開始之前

- **選擇大膽且與內容相關的色彩方案**：色彩方案應該讓人感覺是為「這個」主題量身設計的。如果把你的配色換到一個完全不同的簡報中仍然「適用」，那代表你的選擇還不夠具體。
- **主導而非均等**：一個顏色應佔主導地位（60-70% 視覺比重），搭配 1-2 個輔助色調和一個鮮明的強調色。絕不要讓所有顏色權重相等。
- **深淺對比**：標題與結論投影片使用深色背景，內容頁使用淺色（「三明治」結構）。或者全程使用深色背景以營造高級感。
- **堅持一個視覺主題**：選擇一個獨特的元素並重複使用——圓角圖片框、彩色圓圈中的圖示、粗單邊邊框。在每張投影片中貫徹使用。

### 色彩方案

選擇與主題相符的顏色——不要預設使用通用藍色。以下色彩方案供參考：

| 主題 | 主色 | 輔色 | 強調色 |
|------|------|------|--------|
| **午夜行政** | `1E2761` (navy) | `CADCFC` (ice blue) | `FFFFFF` (white) |
| **森林與苔蘚** | `2C5F2D` (forest) | `97BC62` (moss) | `F5F5F5` (cream) |
| **珊瑚活力** | `F96167` (coral) | `F9E795` (gold) | `2F3C7E` (navy) |
| **暖陶土** | `B85042` (terracotta) | `E7E8D1` (sand) | `A7BEAE` (sage) |
| **海洋漸層** | `065A82` (deep blue) | `1C7293` (teal) | `21295C` (midnight) |
| **炭灰極簡** | `36454F` (charcoal) | `F2F2F2` (off-white) | `212121` (black) |
| **青綠信任** | `028090` (teal) | `00A896` (seafoam) | `02C39A` (mint) |
| **莓果與奶油** | `6D2E46` (berry) | `A26769` (dusty rose) | `ECE2D0` (cream) |

### 每張投影片

**每張投影片都需要一個視覺元素**——圖片、圖表、圖示或形狀。純文字投影片令人難忘。

**版面選項：**
- 雙欄（左側文字，右側插圖）
- 圖示 + 文字列（彩色圓圈中的圖示、粗體標題、下方描述）
- 2x2 或 2x3 網格（一側圖片，另一側內容區塊網格）
- 半出血圖片（完整左側或右側）搭配內容疊加

**資料呈現：**
- 大型統計數字標註（60-72pt 大數字搭配下方小標籤）
- 比較欄位（前後對比、優缺點、並排選項）
- 時間軸或流程圖（編號步驟、箭頭）

**視覺修飾：**
- 段落標題旁的小彩色圓圈圖示
- 關鍵統計數字或標語使用斜體強調文字

### 字型排版

**選擇有趣的字型搭配**——不要預設使用 Arial。選擇一個有個性的標題字型，搭配乾淨的內文字型。

| 標題字型 | 內文字型 |
|----------|----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Calibri | Calibri Light |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| 元素 | 大小 |
|------|------|
| 投影片標題 | 36-44pt 粗體 |
| 段落標題 | 20-24pt 粗體 |
| 內文 | 14-16pt |
| 說明文字 | 10-12pt 淡色 |

### 間距

- 最小邊距 0.5"
- 內容區塊間距 0.3-0.5"
- 留出呼吸空間——不要填滿每一寸

### 避免（常見錯誤）

- **不要重複相同版面**——在投影片之間變化欄位、卡片和標註
- **不要置中內文**——段落和列表靠左對齊；僅標題置中
- **不要忽略大小對比**——標題需要 36pt 以上才能從 14-16pt 內文中脫穎而出
- **不要預設使用藍色**——選擇反映特定主題的顏色
- **不要隨意混用間距**——選擇 0.3" 或 0.5" 間距並一致使用
- **不要只美化一張投影片而其餘保持素面**——要麼全面投入，要麼保持簡潔統一
- **不要製作純文字投影片**——加入圖片、圖示、圖表或視覺元素；避免純標題 + 條列
- **不要忘記文字框內距**——當對齊線條或形狀與文字邊緣時，將文字框的 `margin: 0` 或偏移形狀以補償內距
- **不要使用低對比度元素**——圖示和文字都需要與背景有強烈對比；避免淺色背景上的淺色文字或深色背景上的深色文字
- **絕對不要在標題下方使用強調線**——這是 AI 生成投影片的典型特徵；改用留白或背景色

## 品質檢查（必要）

**假設存在問題。你的工作是找出它們。**

你的第一次渲染幾乎不會是正確的。將品質檢查視為抓蟲行動，而非確認步驟。如果第一次檢查就找不到任何問題，代表你檢查得不夠仔細。

### 內容品質檢查

```bash
python -m markitdown output.pptx
```

檢查是否有遺漏內容、錯字、順序錯誤。

**使用範本時，檢查是否有殘留的佔位文字：**

```bash
python -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|this.*(page|slide).*layout"
```

如果 grep 有回傳結果，在宣告完成之前修正它們。

### 視覺品質檢查

**⚠️ 使用子代理**——即使只有 2-3 張投影片。你一直盯著程式碼，會看到你預期的結果，而非實際的結果。子代理有全新的視角。

將投影片轉換為圖片（參見[轉換為圖片](#轉換為圖片)），然後使用以下提示：

```
視覺檢查這些投影片。假設存在問題——找出它們。

檢查項目：
- 元素重疊（文字穿過形狀、線條穿過文字、元素堆疊）
- 文字溢出或在邊緣／框線處被截斷
- 裝飾線為單行文字定位但標題換行為兩行
- 來源引用或頁尾與上方內容碰撞
- 元素間距過近（< 0.3" 間距）或卡片／區塊幾乎相觸
- 間距不均（某處大片空白，另一處擁擠）
- 與投影片邊緣的邊距不足（< 0.5"）
- 欄位或類似元素未一致對齊
- 低對比度文字（例如淺灰色文字在奶油色背景上）
- 低對比度圖示（例如深色圖示在深色背景上且無對比圓圈）
- 文字框過窄導致過度換行
- 殘留的佔位內容

針對每張投影片，列出發現的問題或疑慮，即使是次要的。

讀取並分析這些圖片：
1. /path/to/slide-01.jpg（預期：[簡短描述]）
2. /path/to/slide-02.jpg（預期：[簡短描述]）

回報所有發現的問題，包括次要問題。
```

### 驗證循環

1. 產生投影片 → 轉換為圖片 → 檢查
2. **列出發現的問題**（如果沒有發現問題，更仔細地再看一次）
3. 修正問題
4. **重新驗證受影響的投影片**——一個修正往往會產生另一個問題
5. 重複直到完整檢查未發現新問題

**在完成至少一次修正與驗證循環之前，不要宣告成功。**

## 轉換為圖片

將簡報轉換為個別投影片圖片以進行視覺檢查：

```bash
python scripts/office/soffice.py --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
```

這會建立 `slide-01.jpg`、`slide-02.jpg` 等檔案。

修正後重新渲染特定投影片：

```bash
pdftoppm -jpeg -r 150 -f N -l N output.pdf slide-fixed
```

## 相依套件

- `pip install "markitdown[pptx]"` - 文字擷取
- `pip install Pillow` - 縮圖網格
- `npm install -g pptxgenjs` - 從零開始建立
- LibreOffice (`soffice`) - PDF 轉換（透過 `scripts/office/soffice.py` 自動設定沙盒環境）
- Poppler (`pdftoppm`) - PDF 轉圖片
