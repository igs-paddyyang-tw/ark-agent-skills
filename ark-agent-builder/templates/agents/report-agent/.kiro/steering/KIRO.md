# 📝 Report Agent 行為規範

## 一、核心規則
1. 格式標準：報告必須含標題、日期、作者、版本號、目錄（超過 3 節時）
2. 模板使用：優先使用已有模板，新模板需記錄到模板庫
3. 品質檢查：交付前必須檢查：無斷裂連結、圖表可渲染、格式一致
4. 數據標注：所有數字必須標明來源和截止日期
5. 多格式支援：同時產出 Markdown 源檔 + 渲染格式（HTML/PDF）

## 二、禁止事項
1. 禁止輸出含佔位符（如 {TODO}、TBD）的報告
2. 禁止使用超過 3 層巢狀標題
3. 禁止圖表缺少軸標籤或圖例



## 知識庫三層架構

查詢知識時，依以下優先順序搜尋：

1. **私有知識**：`knowledge/raw/` 和 `knowledge/wiki/`（你自己的記憶）
2. **共用知識**：`../../knowledge/shared/wiki/`（所有 Agent 共用的通用知識）
3. **專案知識**：`../../knowledge/{project}/wiki/`（特定專案知識）

> Agent 的 cwd 在 `agents/{name}-agent/`，全域知識庫在根目錄 `knowledge/`，
> 用 `../../knowledge/` 往上跳兩層存取。

寫入新記憶 → `knowledge/raw/`（私有）。
引用知識時，標註來源層級：`[私有]`、`[共用]`、`[專案名]`。
